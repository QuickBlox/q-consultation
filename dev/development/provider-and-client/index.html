  • Navigational library: React Router
  • Managing application state: Redux, Redux-Saga
  • Internationalization libraries: i18next, react-i18next
  • Date library: Moment.js
  • Utility library: Lodash
  • Code Linting: Prettier, ESLint
  • Application bundler: Webpack

Screens, Modules and Componentsâ€‹

UI is divided into 3 parts: Screens, Modules and Components. Each of this part is a React component and includes JSX (index.tsx), styles (styles.css) and functional part (useComponent.ts).

Components are the smallest UI parts of an application. These include only reusable components that do not have global state (Redux Store) and do not affect the business logic of the application. They get all the necessary data through props.

Modules can also be reusable, but they already have access to the global state or affect the business logic of the proposal.

Screens are essentially application pages. They can combine several modules and also have access to the global state and can affect the business logic of the application.

Routingâ€‹

Routing and navigation through the application is implemented using the React Router library.

The main part of the routing logic is implemented in the RootRoute screen (src/screens/index.tsx), where the QuickBlox SDK is initialized and the application's public and private routes are separated. All private routes are located in the RootScreen and contain the bulk of the application's business logic. Public routes represent the authorization and registration of users.

Reduxâ€‹

Redux is a structured approach for storing and updating application state.

The Redux part of the app consists of Store, Actions, Action creators, Reducers and Sagas.

Store is the global store/state of the application.

Actions are simple objects that represent a payload that sends data from the application to the store.

An action creator is a function that returns an action object.

Reducer is a pure function that takes the previous state and action as arguments and returns the new state. Reducers define how the application's state changes in response to actions sent to the store.

Sagas are generator functions that allow you to process actions. With the help of them, the main part of the business logic of the application is implemented.

Internationalizationâ€‹

The react-i18next library was used to implement the translations.

Translations can be found in src/translations directory in every packages.â€‹

You can add text here with new languages. After adding new languages, be sure to include them in the i18next configuration files (src/i18n.ts).
